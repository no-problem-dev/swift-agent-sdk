# ユーザーストーリー

## FF-001: CLI プロセス管理

### US-001: CLI 自動検出
- **As a** Swift 開発者
- **I want to** SDK が CLI バイナリの場所を自動的に検出してくれる
- **So that** パスを手動で指定しなくてもすぐに使い始められる

### US-002: CLI パス明示指定
- **As a** CI/CD 環境で利用する開発者
- **I want to** CLI バイナリのパスを明示的に指定できる
- **So that** 非標準のインストール先でも確実に動作する

### US-003: ランタイム選択
- **As a** Bun を好む開発者
- **I want to** Node.js 以外のランタイム（Bun, Deno）を指定できる
- **So that** 好みの JS ランタイムで CLI を実行できる

---

## FF-004: ワンショットクエリ

### US-004: 基本的なプロンプト送信
- **As a** Swift CLI ツール開発者
- **I want to** プロンプト文字列を渡して Claude の応答をストリーミングで受け取れる
- **So that** AI 機能を自分のツールに組み込める

### US-005: モデル指定
- **As a** 開発者
- **I want to** 使用するモデル（opus, sonnet, haiku）を指定できる
- **So that** コストと品質のバランスを制御できる

### US-006: ツールフィルタリング
- **As a** セキュリティ意識の高い開発者
- **I want to** エージェントが使えるツールを許可リスト/拒否リストで制限できる
- **So that** 意図しないファイル操作やコマンド実行を防げる

### US-007: 作業ディレクトリ指定
- **As a** 開発者
- **I want to** エージェントの作業ディレクトリを指定できる
- **So that** 特定のプロジェクトをコンテキストとして操作させられる

### US-008: ターン数・コスト制限
- **As a** コスト管理をしたい開発者
- **I want to** 最大ターン数やコスト上限を設定できる
- **So that** 予期しない長時間実行やコスト超過を防げる

### US-009: システムプロンプト設定
- **As a** カスタムエージェント構築者
- **I want to** システムプロンプトをカスタム文字列またはプリセットから選べる
- **So that** エージェントの振る舞いをユースケースに合わせて調整できる

---

## FF-005: セッション管理

### US-010: セッション作成
- **As a** 対話型アプリ開発者
- **I want to** セッションを作成して複数回のメッセージをやり取りできる
- **So that** コンテキストを維持した対話ができる

### US-011: セッション再開
- **As a** 開発者
- **I want to** セッション ID で以前のセッションを再開できる
- **So that** 中断した作業を続けられる

### US-012: コールドスタート回避
- **As a** レスポンス速度を重視する開発者
- **I want to** 同一セッション内での追加メッセージ送信がコールドスタートなしで行える
- **So that** 初回以降の応答が速い

---

## FF-006: 権限ハンドリング

### US-013: 権限モード設定
- **As a** 開発者
- **I want to** 権限モード（default, acceptEdits, bypassPermissions, plan）を設定できる
- **So that** エージェントの自律度を制御できる

### US-014: カスタム権限ハンドラ
- **As a** 精密な権限制御をしたい開発者
- **I want to** ツールごとにカスタムの許可/拒否ロジックを定義できる
- **So that** ツール実行の可否をプログラムで判断できる

---

## FF-007: サブエージェント定義

### US-015: サブエージェント設定
- **As a** マルチエージェントシステム構築者
- **I want to** 名前付きのサブエージェントを定義してクエリに渡せる
- **So that** 専門エージェントに特定のタスクを委譲できる

---

## FF-008: MCP サーバー設定

### US-016: MCP サーバー構成
- **As a** MCP ツールを利用したい開発者
- **I want to** MCP サーバーの設定（コマンド、引数、環境変数）を渡せる
- **So that** カスタムツールをエージェントに提供できる

---

## FF-009: ランタイム制御

### US-017: クエリ中断
- **As a** 開発者
- **I want to** 実行中のクエリを中断できる
- **So that** 不要な処理を即座に止められる

### US-018: ランタイムモデル変更
- **As a** 開発者
- **I want to** クエリ実行中にモデルを切り替えられる
- **So that** 途中でコスト/品質のバランスを調整できる

---

## FF-010: エラーハンドリング

### US-019: エラー分類
- **As a** 開発者
- **I want to** エラーが分類されて返ってくる（プロセスエラー、プロトコルエラー、タイムアウト等）
- **So that** エラー種別に応じた適切なハンドリングができる

### US-020: CLI 未検出エラー
- **As a** 初めて利用する開発者
- **I want to** CLI が見つからない場合に、インストール方法を含む分かりやすいエラーメッセージが出る
- **So that** セットアップに迷わない

---

## FF-011: プロトコル指向設計 + DI

### US-021: プロトコルに対するプログラミング
- **As a** SDK 利用者
- **I want to** 具象型（ClaudeCodeClient）ではなくプロトコル型（AgentClient）に対してコードを書ける
- **So that** バックエンドの変更やテスト時のモック差し替えで既存コードを変更しなくて済む

### US-022: Transport の DI
- **As a** SDK 利用者
- **I want to** Client 生成時に Transport 実装をコンストラクタで注入できる
- **So that** Claude Code CLI 以外のバックエンド（モック、将来の Direct API 等）に切り替えられる

### US-023: テスト用モックの利用
- **As a** テストコードを書く開発者
- **I want to** AgentTransport / AgentClient / AgentSession のモック実装を簡単に作れる
- **So that** Claude Code CLI を起動せずにユニットテストが書ける

### US-024: デフォルト具象の簡便利用
- **As a** 手早く使い始めたい開発者
- **I want to** DI を意識せずにデフォルトの ClaudeCode 実装でワンライナーで使える
- **So that** プロトコル指向設計の複雑さを感じずに最速で動かせる

### US-025: Claude Code SDK 更新追従
- **As a** SDK メンテナー
- **I want to** Claude Code CLI のプロトコル変更時に具象層（ClaudeCode*）のみ修正すれば済む
- **So that** プロトコル層の安定性が保たれ、利用者のコードが壊れない

### US-026: 将来のバックエンド拡張
- **As a** SDK 拡張者
- **I want to** AgentTransport に準拠した新しいバックエンド（例: Direct API、他の LLM プロバイダ）を追加できる
- **So that** SDK のプロトコル定義を再利用して異なるバックエンドをサポートできる
