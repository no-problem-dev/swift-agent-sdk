# 機能要件

## FF-001: CLI プロセス管理

### FR-001: CLI バイナリ自動探索
- **説明:** SDK は以下の順序で CLI バイナリを探索する
  1. ユーザー指定パス（オプション）
  2. 環境変数 `CLAUDE_CODE_CLI_PATH`
  3. プロジェクト内 `node_modules/@anthropic-ai/claude-agent-sdk/cli.js`
  4. グローバル npm パッケージ
  5. システム PATH 上の `claude` コマンド
- **受け入れ条件:**
  - 正常系: 上記のいずれかで CLI が見つかった場合、そのパスが返る
  - 異常系: すべて不在の場合、検索したパス一覧を含むエラーが発生する
  - 境界値: パスが存在するが実行権限がない場合、権限エラーが発生する

### FR-002: CLI プロセス起動
- **説明:** SDK は指定されたランタイム（node/bun/deno）で CLI プロセスを起動する
- **受け入れ条件:**
  - 正常系: プロセスが起動し、stdin/stdout パイプが確立される
  - 異常系: ランタイムが見つからない場合、エラーが発生する
  - 異常系: プロセス起動に失敗した場合、エラーが発生する

### FR-003: CLI プロセス引数構成
- **説明:** SDK は CLI に以下の引数を渡して起動する
  - `--output-format stream-json`
  - `--input-format stream-json`
  - `--verbose`
  - `--system-prompt`（指定時）
  - `--permission-mode`（指定時）
  - `--setting-sources`（指定時）
- **受け入れ条件:**
  - 正常系: オプションに応じた引数が正しく構成される
  - 境界値: オプション未指定時はデフォルト値が使用される

### FR-004: CLI プロセス終了検知
- **説明:** SDK は CLI プロセスの異常終了を検知し、エラーとして伝播する
- **受け入れ条件:**
  - 正常系: プロセスが exit code 0 で終了した場合、ストリームが正常に完了する
  - 異常系: 非ゼロ exit code の場合、exit code と stderr を含むエラーが発生する
  - 異常系: プロセスがシグナルで kill された場合、対応するエラーが発生する

### FR-005: CLI プロセスクリーンアップ
- **説明:** SDK オブジェクトの破棄時またはキャンセル時に CLI プロセスを終了する
- **受け入れ条件:**
  - 正常系: SDK のライフサイクル終了時にプロセスが terminate される
  - 正常系: Swift Task がキャンセルされた場合、プロセスが terminate される
  - 異常系: プロセスが terminate に応答しない場合、SIGKILL が送信される

## FF-002: JSONL トランスポート

### FR-006: JSONL エンコード（送信）
- **説明:** SDK は構造化メッセージを JSON 文字列 + 改行として stdin に書き込む
- **受け入れ条件:**
  - 正常系: JSON 文字列が 1 行（改行なし）で出力され、末尾に `\n` が付く
  - 境界値: 空のオブジェクト `{}` が正しくエンコードされる
  - 異常系: エンコード不可能な値がある場合、エラーが発生する

### FR-007: JSONL デコード（受信）
- **説明:** SDK は stdout から行単位で JSON を読み取り、型付きメッセージにデコードする
- **受け入れ条件:**
  - 正常系: 各行が独立した JSON オブジェクトとしてパースされる
  - 異常系: 不正な JSON 行を受信した場合、プロトコルエラーが発生する
  - 境界値: 空行は無視される

### FR-008: メッセージタイプ識別
- **説明:** SDK は受信メッセージの `type` フィールドで以下を識別する
  - `initialize_ready`
  - `assistant`（+ `subtype` で partial / complete を区別）
  - `user`
  - `result`
  - `system`
  - `control_request`
  - `control_response`
- **受け入れ条件:**
  - 正常系: 既知のタイプが正しく識別・ルーティングされる
  - 異常系: 未知のタイプは無視される（将来の拡張に対する前方互換性）

### FR-009: stderr キャプチャ
- **説明:** SDK は CLI の stderr 出力をキャプチャし、エラー情報として保持する
- **受け入れ条件:**
  - 正常系: stderr の内容がエラー発生時に参照可能である
  - 境界値: stderr が空の場合でもエラーにならない

## FF-003: 初期化ハンドシェイク

### FR-010: initialize_ready 待機
- **説明:** プロセス起動後、CLI からの `{"type": "initialize_ready"}` を待つ
- **受け入れ条件:**
  - 正常系: 60 秒以内に受信した場合、初期化を続行する
  - 異常系: 60 秒以内に受信しない場合、タイムアウトエラーが発生する
  - 異常系: `initialize_ready` より前に他のメッセージが来た場合、プロトコルエラーが発生する

### FR-011: InitializeRequest 送信
- **説明:** `initialize_ready` 受信後、hook 登録等を含む初期化リクエストを送信する
- **受け入れ条件:**
  - 正常系: 初期化リクエストが正しい形式で stdin に送信される
  - 正常系: hooks が指定されている場合、hook マッチャーが含まれる

### FR-012: SystemMessage 受信
- **説明:** 初期化完了後、session_id・ツール一覧・モデル情報を含む SystemMessage を受信する
- **受け入れ条件:**
  - 正常系: session_id がストリーム最初のメッセージとして返される
  - 正常系: 利用可能ツール一覧が含まれる

## FF-004: ワンショットクエリ

### FR-013: query() 関数
- **説明:** プロンプト文字列とオプションを受け取り、メッセージの AsyncSequence を返す
- **受け入れ条件:**
  - 正常系: プロンプトを送信し、ストリーミングメッセージが返される
  - 正常系: オプション未指定時はデフォルト値が使用される
  - 異常系: CLI 起動に失敗した場合、エラーが throw される

### FR-014: ストリーミングメッセージ配信
- **説明:** CLI からのメッセージをリアルタイムで AsyncSequence に yield する
- **受け入れ条件:**
  - 正常系: assistant メッセージ、partial メッセージ、result メッセージが順序通りに配信される
  - 正常系: result メッセージの受信で AsyncSequence が完了する
  - 異常系: プロセス異常終了時に AsyncSequence がエラーで完了する

### FR-015: オプション: model 指定
- **説明:** `model` オプションで使用するモデルを指定できる
- **受け入れ条件:**
  - 正常系: 指定したモデルが CLI に渡される
  - 境界値: 未指定時はデフォルトモデルが使用される

### FR-016: オプション: allowedTools / disallowedTools
- **説明:** エージェントが使用可能なツールをフィルタリングできる
- **受け入れ条件:**
  - 正常系: allowedTools 指定時、リスト内のツールのみ使用可能
  - 正常系: disallowedTools 指定時、リスト内のツールが使用不可
  - 境界値: 両方未指定時は全ツール使用可能

### FR-017: オプション: 構造化出力
- **説明:** `outputFormat` に JSON Schema を指定して、構造化された応答を要求できる
- **受け入れ条件:**
  - 正常系: JSON Schema が CLI に渡され、応答が指定形式に従う
  - 境界値: 未指定時は自由形式のテキスト応答

## FF-005: セッション管理

### FR-018: セッション作成
- **説明:** 新規セッションを作成し、複数回のメッセージ交換を可能にする
- **受け入れ条件:**
  - 正常系: セッションオブジェクトが返され、session_id が取得できる
  - 正常系: CLI プロセスがセッション期間中維持される

### FR-019: セッション内メッセージ送信
- **説明:** 既存セッションにメッセージを送信し、応答ストリームを受け取る
- **受け入れ条件:**
  - 正常系: コールドスタートなしでメッセージが処理される
  - 正常系: 前回の会話コンテキストが維持される

### FR-020: セッション再開
- **説明:** 保存した session_id で以前のセッションを再開する
- **受け入れ条件:**
  - 正常系: 再開後、以前のコンテキストを持った状態で会話が続行される
  - 異常系: セッションが期限切れの場合、エラーが発生する

### FR-021: セッション ID 取得
- **説明:** セッションオブジェクトから session_id を取得できる
- **受け入れ条件:**
  - 正常系: 初期化完了後、一意の session_id が取得可能

### FR-022: セッション終了
- **説明:** セッションを明示的に終了し、CLI プロセスをクリーンアップする
- **受け入れ条件:**
  - 正常系: CLI プロセスが正常に終了する
  - 正常系: 以降のメッセージ送信はエラーとなる

## FF-006: 権限ハンドリング

### FR-023: 権限モード設定
- **説明:** 以下の権限モードを設定できる
  - `default` - 各ツール使用時に確認
  - `acceptEdits` - ファイル編集を自動承認
  - `bypassPermissions` - 全ツールを自動承認
  - `plan` - 計画モード
- **受け入れ条件:**
  - 正常系: 指定した権限モードが CLI に渡される
  - 境界値: 未指定時は `default` が使用される

### FR-024: カスタム権限ハンドラ
- **説明:** ツール使用リクエストに対してカスタムの許可/拒否を返すクロージャを設定できる
- **受け入れ条件:**
  - 正常系: CLI からの `can_use_tool` リクエスト受信時にクロージャが呼ばれる
  - 正常系: クロージャの戻り値（allow/deny）が CLI に返送される
  - 境界値: クロージャ未設定時は権限モードに従う

### FR-025: 権限リクエスト情報
- **説明:** カスタム権限ハンドラにはツール名、入力パラメータ、サーバー情報が渡される
- **受け入れ条件:**
  - 正常系: ツール名が文字列として渡される
  - 正常系: 入力パラメータが辞書として渡される

### FR-026: 権限レスポンスタイプ
- **説明:** 権限の判定結果は以下のいずれか
  - `allow` - 許可
  - `deny(reason:)` - 拒否（理由付き）
- **受け入れ条件:**
  - 正常系: allow の場合、ツールが実行される
  - 正常系: deny の場合、理由が CLI に伝えられツール実行がスキップされる

### FR-027: ランタイム権限モード変更
- **説明:** クエリ/セッション実行中に権限モードを変更できる
- **受け入れ条件:**
  - 正常系: 変更後のモードが即座に適用される

### FR-028: エージェント定義構造
- **説明:** サブエージェントは以下の属性で定義する
  - 名前（キー）
  - 説明文
  - システムプロンプト
  - 使用可能ツール一覧
  - モデル指定（opus/sonnet/haiku/inherit）
- **受け入れ条件:**
  - 正常系: 定義が正しい形式で CLI に渡される
  - 境界値: ツール未指定時はデフォルトセットが使用される

### FR-029: 複数エージェント定義
- **説明:** 1つのクエリ/セッションに複数のサブエージェントを定義できる
- **受け入れ条件:**
  - 正常系: 辞書形式で複数エージェントが渡される

### FR-030: サブエージェントメッセージ識別
- **説明:** サブエージェントからのメッセージは `parent_tool_use_id` で識別できる
- **受け入れ条件:**
  - 正常系: サブエージェントのメッセージに `parent_tool_use_id` が含まれる
  - 正常系: メインエージェントのメッセージには含まれない

- **説明:** CLI がサポートするスラッシュコマンド一覧を取得できる

### FR-038: サポートモデル一覧取得

## FF-010: エラーハンドリング

### FR-039: エラー分類
- **説明:** 以下のエラー種別を区別する
  - CLI 未検出エラー
  - ランタイム未検出エラー
  - プロセス起動失敗エラー
  - プロセス異常終了エラー（exit code + stderr）
  - プロトコルエラー（不正 JSON、予期しないメッセージ）
  - 初期化タイムアウトエラー
  - 権限拒否エラー
- **受け入れ条件:**
  - 正常系: 各エラーが固有の型/ケースとして区別される

### FR-040: エラーメッセージ品質
- **説明:** エラーメッセージには原因と推奨アクションが含まれる
- **受け入れ条件:**
  - 正常系: CLI 未検出時、インストール方法のヒントが含まれる
  - 正常系: プロセスエラー時、exit code と stderr が含まれる

### FR-041: エラー伝播
- **説明:** エラーは Swift の標準的なエラーハンドリング機構（throw）で伝播する
- **受け入れ条件:**
  - 正常系: AsyncSequence のイテレーション中にエラーが throw される
  - 正常系: セッション操作中のエラーが throw される

### FR-042: タイムアウト制御
- **説明:** 初期化・制御リクエストにタイムアウトが設定される
- **受け入れ条件:**
  - 正常系: 初期化タイムアウト: 60 秒
  - 正常系: 制御リクエストタイムアウト: 30 秒
  - 境界値: タイムアウト値のカスタマイズが可能

### FR-043: キャンセレーション対応
- **説明:** Swift Task のキャンセレーションに正しく応答する
- **受け入れ条件:**
  - 正常系: Task.cancel() 呼び出し時に CLI プロセスが終了される
  - 正常系: AsyncSequence のイテレーションが CancellationError で終了する

## FF-011: プロトコル指向設計 + DI

### FR-044: AgentTransport protocol
- **説明:** 通信層の抽象を定義する protocol。バックエンドとの接続・メッセージ送受信・切断を担う
- **要求メソッド:**
  - `connect()` - 接続確立（CLI 起動 + ハンドシェイク相当）
  - `write(_ message:)` - メッセージ送信
  - `messages()` - 受信メッセージの AsyncSequence を返す
  - `close()` - 接続切断（プロセス終了相当）
  - `isReady` - 接続状態の確認
- **受け入れ条件:**
  - 正常系: protocol が `Sendable` に準拠している
  - 正常系: メソッドはすべて `async throws`
  - 正常系: `messages()` は `AsyncThrowingStream` または同等の AsyncSequence を返す
  - 境界値: protocol は Claude Code 固有の概念（JSONL、CLI 等）を含まない

### FR-045: AgentClient protocol
- **説明:** 操作層の抽象を定義する protocol。クエリ・セッション管理を担う
- **要求メソッド:**
  - `query(prompt:options:)` - ワンショットクエリ
  - `createSession(options:)` - セッション作成
  - `resumeSession(id:options:)` - セッション再開
- **受け入れ条件:**
  - 正常系: protocol が `Sendable` に準拠している
  - 正常系: `query()` はメッセージの AsyncSequence を返す
  - 正常系: `createSession()` / `resumeSession()` は `AgentSession` 準拠型を返す（generics / `some AgentSession`）
  - 境界値: protocol は Claude Code 固有の概念を含まない

### FR-046: AgentSession protocol
- **説明:** セッション層の抽象を定義する protocol。セッション内の対話を担う
- **要求メソッド:**
  - `id` - セッション識別子
  - `send(_ message:)` - メッセージ送信・応答ストリーム受信
  - `interrupt()` - 処理中断
  - `close()` - セッション終了
- **受け入れ条件:**
  - 正常系: protocol が `Sendable` に準拠している
  - 正常系: `send()` はメッセージの AsyncSequence を返す
  - 境界値: protocol は Claude Code 固有の概念を含まない

### FR-047: AgentMessage は値型（protocol 非依存）
- **説明:** SDK が返すメッセージ型は特定のバックエンドに依存しない共通の値型として定義する
- **受け入れ条件:**
  - 正常系: AgentMessage は enum または struct で定義される（protocol ではない）
  - 正常系: assistant / result / system 等の種別を含む
  - 正常系: Sendable, Codable に準拠している
  - 境界値: バックエンド固有のフィールド（例: parent_tool_use_id）も共通型に含められる

### FR-048: Transport DI（コンストラクタ注入）
- **説明:** AgentClient の具象型は AgentTransport をコンストラクタで受け取る
- **受け入れ条件:**
  - 正常系: `ClaudeCodeClient(transport: someTransport)` でカスタム Transport を注入できる
  - 正常系: Transport 未指定時はデフォルトの ClaudeCodeTransport が使用される
  - 正常系: テスト時に MockTransport を注入してユニットテストが実行できる

### FR-049: デフォルト具象のコンビニエンス API
- **説明:** DI を意識しない利用者向けに、デフォルトの Claude Code 実装を使うファクトリを提供する
- **受け入れ条件:**
  - 正常系: `AgentSDK.query(prompt:options:)` で DI なしにワンショットクエリが実行できる
  - 正常系: 内部的にデフォルトの ClaudeCodeTransport + ClaudeCodeClient が生成される
  - 境界値: コンビニエンス API は内部で具象型を生成し、generics で型推論される

### FR-050: プロトコル層と具象層のモジュール分離
- **説明:** プロトコル定義と Claude Code 具象実装が分離されていること
- **受け入れ条件:**
  - 正常系: プロトコル層のみを import して型定義・モック実装が可能
  - 正常系: 具象層は別モジュールまたは同一モジュール内で internal 依存がプロトコル層→具象層の方向のみ
  - 境界値: プロトコル層の変更なしに具象層を差し替え可能

### FR-051: Claude Code 具象: ClaudeCodeTransport
- **説明:** AgentTransport に準拠し、Claude Code CLI サブプロセスの JSONL プロトコルを実装する
- **受け入れ条件:**
  - 正常系: FF-001〜FF-003（プロセス管理・JSONL・ハンドシェイク）の全要件を満たす
  - 正常系: Claude Code CLI 固有のロジック（引数構成、メッセージ変換等）がこの型に局所化されている
  - 正常系: CLI バージョン更新時の変更がこの型内に収まる

### FR-052: Claude Code 具象: ClaudeCodeClient
- **説明:** AgentClient に準拠し、ClaudeCodeTransport を使ってクエリ・セッション管理を実装する
- **受け入れ条件:**
  - 正常系: FF-004〜FF-009（クエリ・セッション・権限・サブエージェント等）の全要件を満たす
  - 正常系: Transport は generics パラメータとしてコンストラクタで受け取る（`ClaudeCodeClient<T: AgentTransport>`）
  - 正常系: ClaudeCodeTransport 以外の AgentTransport 準拠型も受け入れ可能

### FR-053: テスト支援: MockTransport
- **説明:** テスト用の AgentTransport 準拠型を SDK が提供する（別モジュール）
- **受け入れ条件:**
  - 正常系: 事前定義したメッセージシーケンスを返す MockTransport が利用可能
  - 正常系: 送信されたメッセージを記録し、テストでアサートできる
  - 境界値: MockTransport は本番コードには含まれない（テストターゲット専用）
