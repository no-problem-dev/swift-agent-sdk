# 機能要件

## FF-001: セッション管理

### FR-001: 新規セッション作成

ユーザーは新規セッションを作成できる。

**正常系:**
- 作業ディレクトリ、モデル、システムプロンプト（任意）を指定して作成する
- 作成後、SDK の `AgentSDK.createSession(options:)` が呼び出される
- セッション ID が SDK から返却され、ローカルに保存される
- 作成されたセッションが自動的にアクティブになる

**異常系:**
- 作業ディレクトリが存在しない場合、エラーメッセージを表示する
- CLI が見つからない場合（`cliNotFound`）、インストール手順を含むアラートを表示する
- 接続タイムアウト時、タイムアウトエラーを表示しセッションをエラー状態にする

**境界値:**
- 作業ディレクトリにスペースや日本語を含むパスが指定された場合、正常に動作する

### FR-002: セッション一覧表示

ユーザーはサイドバーに全セッションの一覧を確認できる。

**正常系:**
- 各セッションに以下が表示される: セッション名、作成日時、モデル、ステータス
- セッション名は最初のユーザーメッセージの先頭 30 文字をデフォルトとする
- 一覧は最終アクティブ日時の降順でソートされる

**異常系:**
- セッションが 0 件の場合、空状態のプレースホルダーを表示する

### FR-003: セッション切替

ユーザーはサイドバーのセッションをクリックしてアクティブセッションを切り替えられる。

**正常系:**
- クリックしたセッションがアクティブになる
- チャットエリアにそのセッションのメッセージ履歴が表示される
- 切替元セッションの SDK 接続は維持される（閉じない）

### FR-004: セッション再開

ユーザーは切断済みセッションを選択して再接続できる。

**正常系:**
- 切断済みセッション選択時、`AgentSDK.resumeSession(id:options:)` が呼び出される
- 再接続中はステータスが「接続中」になる
- 成功時はステータスが「接続済み」に遷移する

**異常系:**
- セッション期限切れ（`sessionExpired`）時、エラーメッセージとともに新規作成を促す
- その他のエラー時、エラーメッセージを表示しステータスを「エラー」にする

### FR-005: セッション終了

ユーザーはセッションの SDK 接続を終了できる。

**正常系:**
- `session.close()` が呼び出される
- ステータスが「切断済み」に遷移する
- セッション一覧からは削除されない（再開可能）

### FR-006: セッション削除

ユーザーはセッションを完全に削除できる。

**正常系:**
- 確認ダイアログで承認後、セッションデータとメッセージ履歴を削除する
- 接続中のセッションは先に `close()` を呼び出してから削除する

**異常系:**
- 確認ダイアログでキャンセルした場合、何もしない

### FR-007: セッション名変更

ユーザーはセッションの表示名を変更できる。

**正常系:**
- コンテキストメニューまたはダブルクリックで名前変更モードに入る
- 変更後の名前がローカルに保存される

## FF-002: チャットメッセージング

### FR-008: メッセージ送信

ユーザーはテキストメッセージを送信できる。

**正常系:**
- テキスト入力後 Enter で `session.send(message)` が呼び出される
- 送信されたメッセージがチャットエリアにユーザーバブルとして表示される
- 送信後、入力欄がクリアされる

**異常系:**
- セッション未接続時にメッセージ送信を試みた場合、エラーを表示する
- 空文字列は送信できない（送信ボタンが無効化される）

**境界値:**
- 改行を含む複数行メッセージが送信できる（Shift+Enter で改行）

### FR-009: ストリーミング表示

応答中の部分テキストをリアルタイムで表示する。

**正常系:**
- `.partial` メッセージ受信ごとにチャットエリアのテキストを更新する
- 新しいテキストが追加されるとチャットエリアが自動スクロールする
- ストリーミング中は送信ボタンが「停止」ボタンに変わる

### FR-010: アシスタント応答表示

完成したアシスタント応答を表示する。

**正常系:**
- `.assistant` メッセージの各 `ContentBlock` を種別に応じて表示する:
  - `.text(String)` → Markdown レンダリング
  - `.toolUse(ToolUse)` → ツール使用カード（FF-003 で定義）
  - `.toolResult(ToolResult)` → ツール結果カード（FF-003 で定義）
- ストリーミング中のテキストは完成メッセージに置き換わる

### FR-011: 処理中断

ユーザーはストリーミング中の処理を中断できる。

**正常系:**
- 停止ボタン押下で `session.interrupt()` が呼び出される
- 中断時点までの応答テキストは保持される
- 入力欄が再び有効化される

**異常系:**
- interrupt 呼び出しに失敗した場合、エラーを表示する

### FR-012: 結果メッセージ処理

`.result` メッセージを受信し、メタデータを更新する。

**正常系:**
- セッションの累計コストに `costUsd` を加算する
- トークン使用量（`inputTokens` / `outputTokens`）を UI に反映する
- 処理中フラグを解除し、入力欄を有効化する

## FF-003: ツール可視化

### FR-013: ツール使用カード

ツール使用（`.toolUse`）を専用カードで表示する。

**正常系:**
- ツール名をヘッダーに表示する（例: "Bash", "Read", "Write"）
- ツールのパラメータ（input）をキー・値形式で表示する
- カードは初期状態で展開されている

### FR-014: ツール結果カード

ツール結果（`.toolResult`）をカードで表示する。

**正常系:**
- ツール使用カードの直下に結果を表示する
- 結果テキストが長い場合は折りたたみ可能にする
- 折りたたみ/展開をクリックで切り替えられる

**異常系:**
- `isError: true` の場合、カードを赤系の背景色で表示する

### FR-015: ツールペア表示

ツール使用と対応する結果をペアとして表示する。

**正常系:**
- 同一 `toolUseId` の ToolUse と ToolResult をペアにして表示する
- 結果が未着の場合（ストリーミング中）、ローディングインジケータを表示する

## FF-004: モデル・設定制御

### FR-016: モデル選択

ユーザーはセッション中にモデルを変更できる。

**正常系:**
- ツールバーに Opus / Sonnet / Haiku のドロップダウンがある
- 変更時に `session.setModel(model)` が呼び出される
- 現在のモデルが常にドロップダウンに反映される

**異常系:**
- setModel に失敗した場合、エラーを表示し選択を元に戻す

### FR-017: 作業ディレクトリ表示

ユーザーは現在のセッションの作業ディレクトリを確認できる。

**正常系:**
- ツールバーに作業ディレクトリのパスが表示される
- パスが長い場合は省略表示（末尾のディレクトリ名 + ツールチップで全パス）

### FR-018: システムプロンプト設定

ユーザーはセッション作成時にシステムプロンプトを設定できる。

**正常系:**
- 新規セッション作成ダイアログにシステムプロンプト入力欄がある
- 未入力の場合は SDK デフォルト（Claude Code 標準）が使用される

### FR-019: コスト表示

ユーザーはセッションの累計コストを確認できる。

**正常系:**
- ツールバーまたはステータス領域に累計コスト（USD）が表示される
- `.result` メッセージ受信時に更新される
- 小数点以下 4 桁まで表示（例: $0.0123）

### FR-020: トークン使用量表示

ユーザーは直近ターンのトークン使用量を確認できる。

**正常系:**
- 入力トークン数と出力トークン数を表示する
- `.result` メッセージ受信時に更新される

## FF-005: データ永続化

### FR-021: セッション情報の保存

セッション情報をローカルに保存する。

**正常系:**
- セッション作成時に以下を保存: セッション ID、設定、作成日時
- 保存先: `~/Library/Application Support/ClaudeAgent/sessions.json`

### FR-022: メッセージ履歴の保存

メッセージ履歴をローカルに保存する。

**正常系:**
- `.assistant` メッセージと送信メッセージを保存する
- `.partial` メッセージは保存しない
- 保存タイミング: `.result` メッセージ受信時（ターン完了時）

### FR-023: データ復元

アプリ起動時にローカルデータを復元する。

**正常系:**
- 起動時に `sessions.json` を読み込み、セッション一覧を復元する
- 各セッションのメッセージ履歴を復元する
- セッションのステータスは「切断済み」として復元される

**異常系:**
- ファイルが存在しない場合、空のセッション一覧で起動する
- ファイルが破損している場合、エラーログを出力し空の状態で起動する

### FR-024: データ削除

セッション削除時にローカルデータを削除する。

**正常系:**
- 該当セッションの情報とメッセージ履歴を `sessions.json` から削除する
