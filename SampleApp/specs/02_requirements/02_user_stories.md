# ユーザーストーリー

## FF-001: セッション管理

### US-001: 新規セッション作成
**As a** 開発者
**I want to** 作業ディレクトリを指定して新しいセッションを作成する
**So that** そのプロジェクトについて Claude に相談できる

**受け入れ条件:**
- [ ] 「+」ボタンまたは Cmd+N で新規セッション作成ダイアログが開く
- [ ] ディレクトリ選択ダイアログで作業ディレクトリを指定できる
- [ ] モデル（Opus / Sonnet / Haiku）を選択できる
- [ ] オプションでシステムプロンプトを入力できる
- [ ] 作成後、そのセッションが自動的に選択される

### US-002: セッション切替
**As a** 開発者
**I want to** サイドバーでセッションを切り替える
**So that** 複数のプロジェクトについて並行して作業できる

**受け入れ条件:**
- [ ] サイドバーのセッション一覧が表示される
- [ ] セッション名・作成日時・ステータスが一覧に表示される
- [ ] クリックでセッションが切り替わる
- [ ] 切替後、そのセッションのメッセージ履歴が表示される

### US-003: セッション再開
**As a** 開発者
**I want to** アプリ再起動後に前回のセッションを再開する
**So that** 前回の会話の続きから作業できる

**受け入れ条件:**
- [ ] 切断済みセッションを選択すると再接続が試みられる
- [ ] 再接続中はローディング表示がある
- [ ] 成功時、前回の会話コンテキストが維持される
- [ ] 失敗時（セッション期限切れ等）、エラーメッセージが表示される
- [ ] 失敗時、新規セッション作成が促される

### US-004: セッション終了
**As a** 開発者
**I want to** 不要なセッションを終了する
**So that** リソースを解放できる

**受け入れ条件:**
- [ ] コンテキストメニューまたはショートカットでセッション終了を選択できる
- [ ] 終了後、セッション一覧にはステータス「切断済み」として残る
- [ ] 終了したセッションは後から再開できる

### US-005: セッション削除
**As a** 開発者
**I want to** 不要なセッションを完全に削除する
**So that** セッション一覧を整理できる

**受け入れ条件:**
- [ ] コンテキストメニューから削除を選択できる
- [ ] 確認ダイアログが表示される
- [ ] 削除後、セッション一覧から消える
- [ ] ローカルのメッセージ履歴データも削除される

## FF-002: チャットメッセージング

### US-006: メッセージ送信
**As a** 開発者
**I want to** テキストメッセージを入力して Claude に送信する
**So that** コードについて質問・相談できる

**受け入れ条件:**
- [ ] 画面下部にテキスト入力欄がある
- [ ] Enter キーで送信される
- [ ] Shift+Enter で改行できる
- [ ] 送信したメッセージがチャットエリアに表示される
- [ ] 空メッセージは送信できない

### US-007: ストリーミング表示
**As a** 開発者
**I want to** Claude の応答がリアルタイムで表示される
**So that** 応答完了を待たずに内容を確認できる

**受け入れ条件:**
- [ ] 応答テキストが逐次追加表示される
- [ ] 表示中も UI がフリーズしない
- [ ] 新しいテキストが追加されると自動スクロールする
- [ ] ストリーミング中は入力欄が無効化される

### US-008: 応答表示
**As a** 開発者
**I want to** Claude の応答を読みやすい形式で見る
**So that** コードブロックや説明を正しく理解できる

**受け入れ条件:**
- [ ] テキスト応答が Markdown としてレンダリングされる
- [ ] コードブロックがシンタックスハイライト付きで表示される
- [ ] ユーザーメッセージとアシスタント応答が視覚的に区別される

### US-009: 処理中断
**As a** 開発者
**I want to** Claude の処理を中断する
**So that** 不要な応答生成を止められる

**受け入れ条件:**
- [ ] ストリーミング中に「停止」ボタンが表示される
- [ ] ボタン押下で処理が中断される
- [ ] 中断後、通常の入力状態に戻る
- [ ] 中断時点までの応答は保持される

## FF-003: ツール可視化

### US-010: ツール使用の確認
**As a** 開発者
**I want to** Claude がどのツールを使ったか確認する
**So that** Claude の動作を理解し信頼できる

**受け入れ条件:**
- [ ] ツール使用が専用カードで表示される
- [ ] ツール名とパラメータが表示される
- [ ] ツール結果が折りたたみ/展開できる
- [ ] エラー結果は視覚的に区別される（赤系色など）

---

## FF-004: モデル・設定制御

### US-011: モデル切替
**As a** 開発者
**I want to** セッション中にモデルを切り替える
**So that** タスクの複雑さに応じて適切なモデルを使える

**受け入れ条件:**
- [ ] ツールバーにモデル選択 UI がある
- [ ] Opus / Sonnet / Haiku から選択できる
- [ ] 選択するとセッション中にリアルタイム反映される
- [ ] 現在のモデルが常に表示されている

### US-012: コスト確認
**As a** 開発者
**I want to** 現在のセッションでかかったコストを確認する
**So that** 利用量を把握できる

**受け入れ条件:**
- [ ] セッションの累計コスト（USD）が表示される
- [ ] ターンごとの入力/出力トークン数が確認できる
- [ ] 情報は `.result` メッセージ受信時に更新される

---

## FF-005: データ永続化

### US-013: メッセージ履歴の保持
**As a** 開発者
**I want to** アプリを閉じてもメッセージ履歴が保持される
**So that** 前回のやり取りを振り返れる

**受け入れ条件:**
- [ ] アプリ再起動後もメッセージ履歴が表示される
- [ ] ユーザーメッセージ・アシスタント応答・ツール使用が保存される
- [ ] ストリーミング途中の partial メッセージは保存されない

### US-014: セッション一覧の保持
**As a** 開発者
**I want to** アプリを閉じてもセッション一覧が保持される
**So that** 前回のセッションに再接続できる

**受け入れ条件:**
- [ ] アプリ再起動後もセッション一覧が表示される
- [ ] 各セッションの設定（cwd, モデル）が復元される
- [ ] 各セッションの累計コストが復元される
