# 非機能要件

## NFR-001: パフォーマンス

| 項目 | 要件 | 測定方法 |
|------|------|---------|
| UI フレームレート | ストリーミング中 30fps 以上を維持 | Instruments の Core Animation |
| partial 反映遅延 | メッセージ受信から UI 反映まで 100ms 以内 | ストリーミング中の主観的なカクつきがないこと |
| アプリ起動時間 | 起動から操作可能まで 3 秒以内（セッション接続除く） | コールドスタートから UI 表示までの時間 |
| メモリ使用量 | 1 セッション（100 メッセージ）で 200MB 以下 | Activity Monitor での RSS |

## NFR-002: 信頼性

| 項目 | 要件 | 測定方法 |
|------|------|---------|
| クラッシュ率 | SDK エラーでアプリがクラッシュしない | 全 AgentSDKError をキャッチし UI 表示する |
| データ損失 | 保存済みデータが不意に失われない | ファイル書き込みはアトミック操作で行う |
| セッション復旧 | エラー後にセッション再開が可能 | エラー状態からの再接続テスト |

## NFR-003: ユーザビリティ

| 項目 | 要件 | 測定方法 |
|------|------|---------|
| ダークモード | macOS のシステム設定に追従する | ライトモード・ダークモードの両方で視認性確保 |
| キーボードショートカット | 主要操作をキーボードで実行可能 | Cmd+N（新規）、Enter（送信）、Esc（中断） |
| エラーメッセージ | エラー内容をユーザーが理解できる言葉で表示 | 技術的エラーをユーザー向けメッセージに変換 |
| 接続状態表示 | セッションの接続状態が常に視認可能 | ステータスバッジまたはアイコンの視認テスト |

## NFR-004: 保守性

| 項目 | 要件 | 測定方法 |
|------|------|---------|
| Swift 6 準拠 | strict concurrency で警告なし | `SWIFT_STRICT_CONCURRENCY=complete` でビルド |
| MVVM 遵守 | View に業務ロジックを含まない | コードレビューでの確認 |
| テスタビリティ | ViewModel 層は MockTransport でテスト可能 | ユニットテストの存在確認 |
| コード量 | サンプルアプリとして適切な規模（3,000 行以下） | cloc での計測 |

## NFR-005: 互換性

| 項目 | 要件 |
|------|------|
| OS バージョン | macOS 15.0 以上 |
| アーキテクチャ | Apple Silicon (arm64) + Intel (x86_64) |
| Swift バージョン | 6.0 以上 |
| Xcode バージョン | 16.0 以上 |
| swift-agent-sdk | 0.1.x（ローカルパス参照） |
